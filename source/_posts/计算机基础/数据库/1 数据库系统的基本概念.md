# 数据库系统的基本概念
## 数据库和数据库管理系统定义
DB:数据库
DBMS:数据库管理系统
## 数据抽象的三层结构
- 物理层：描述数据实际上是怎么存储的
- 逻辑层：描述数据库中存储什么数据以及这些数据间存在什么关系
- 视图层：应用程序能够隐藏数据类型的详细信息
物理数据独立性和逻辑数据独立性
## 实例和模式
模式（schema）：数据库的总体设计
	物理模式：在物理层描述数据库的设计
	逻辑模式：在逻辑层描述数据库的设计
实例（instance）：特定时刻存储中数据库中的信息集合
## 数据模型
用于描述：数据结构、数据关系、数据语义和数据约束
**Object-based data model（面向对象的数据模型）：**
- Entity-Relational Model（实体关系模型）
- Object-oriented Model（面向对象模型）
- Record-based data model（基于记录的数据模型）
**Record-based data model（基于记录的数据模型）：**
- Hierarchical data model（层次数据模型）
- Network data model（网络数据模型）
- Relational data model（关系数据模型）
## 数据库语言
- Data Definition Language (DDL 数据定义语言)
	- 数据定义语言DDL
		- 指定一个数据库模式作为一组关系模式的定义
		- 指定存储结构，访问方法和一致性约束
		- DDL语言经过编译，得到一组存储在一个特殊文件里的表，特殊文件即数据字典，其中包含元数据。
			- 元数据包括：
				- 数据库模式
				- 数据存储结构
				- 访问方法和约束
				- 统计信息
				- 授权
		- `CREATE TABLE account(account_number char(10),balance integer);`
- Data Manipulation Language (DML 数据操纵语言)
	- 从数据库中检索数据
	- 插入、删除、更新数据
	- DML也称为查询语言
- Data Control Language (DDL 数据定义语言)
	- 两类基本的数据操作语言
		- 过程化DML：用户指定需要什么数据，以及如何获得这些数据（c）
		- 声明式DML：要求用户指定需要什么数据，而不指明如何获得什么数据（sql）
## 数据库设计步骤
1. 需求分析
	需要什么样的数据、应用程序和业务
2. 概念数据库设计
	使用E-R模型或类似的高层次数据模型描述数据
3. 逻辑数据库设计
	将概念设计转换为某个DBMS所支持的数据类型
4. 结构优化
	关系标准化，检查冗余和相关的异常关系结构
5. 物理数据库设计
	索引、集群和数据库调优
6. 创建并初始化数据库&安全设计
	加载初始数据，测试
	识别不同的用户及他们的角色
