# C语言负数整形的除法和取余
<p>C语言整形的除法的结果需要取整&#xff0c;例如<code>5 / 3 &#61; 1</code>。C语言中的取整方式是向零取整。</p> 
<p>取整风格共有3种&#xff1a;</p> 
<ol><li>向零取整。即向坐标轴0的方向&#xff0c;取最近的整数。</li><li>向上取整。也就是向&#43;∞取整&#xff0c;即取不小于结果的最小整数。</li><li>向下取整。也就是向-∞取整&#xff0c;即取不大于结果的最大整数。</li></ol>
<p>所以&#xff0c;C语言中计算-5除以3&#xff0c;因为实数结果约为-1.67&#xff0c;向零取整&#xff0c;得到结果为-1。</p> 
<p>关于除法&#xff0c;人们常常用向右移位的方式来快速计算一个数除以2的幂的结果&#xff0c;例如<code>16 / 4 &#61; 4</code>就是把10000右移2位得到100&#xff0c;即4&#xff1b;<code>9 / 4 &#61; 2</code>就是把1001右移2位得到2。事实上&#xff0c;这种快速计算的方法&#xff0c;是一种向下取整的过程&#xff0c;所以它只适用于被除数为正数的情况。若被除数为负数&#xff0c;这种右移的快速计算方式就是错误的了。下面用代码来验证一下&#xff0c;从10到-10&#xff0c;分别计算他们除以4和右移2位的结果:</p> 

```c
#include <stdio.h>
 
int main()
{
  int i = 0;
 
  for (i = 10; i >= -10; i--)
  {
    printf("%d / 4 = %d\r\n", i, i / 4);
    printf("%d >> 2 = %d\r\n", i, i >> 2);
    printf("\r\n");
  }
 
  getchar();
    
  return 0;
}
```

<p>可见&#xff0c;当被除数为正数的时候&#xff0c;除以4和右移2位的结果是一样的。但是&#xff0c;当被除数为负数的时候&#xff0c;这两个结果并不一定相同。</p> 
<p>关于负数的右移操作的解释&#xff1a;以-6为例&#xff0c;6的二进制形式是110&#xff0c;所以-6的补码形式是第一位符号位为1&#xff0c;数据位是110取反加1&#xff0c;得到1010。1010右移2位&#xff0c;得到10。取出10的符号位1&#xff0c;数据位是0&#xff0c;数据位取反加1得到10&#xff0c;即十进制的2。也就是十进制-6右移2位得到十进制的-2。</p> 
<p>有了除法的基础&#xff0c;取余&#xff08;也叫求模&#xff09;操作根据公式计算就可以了。设A除以B的结果为Q&#xff0c;余数为R&#xff0c;则<code>A &#61; B * Q &#43; R&#xff0c; R &#61; A - B *Q</code>。例如求<code>-8 % 3</code>,因为<code>-8 / 3 &#61; -2</code>&#xff0c;那么余数就是<code>-8 - 3 * (-2) &#61; -2</code>。</p> 
<p>取余操作除了套用公式来做&#xff0c;还有一个快捷的经验&#xff0c;就是取余的结果的符号永远和被除数的符号一样。这个经验可以用下面的例子来解释&#xff1a;去银行借了8万块贷款&#xff0c;每个月还3万&#xff0c;还了2个月之后&#xff0c;还剩下2万没还&#xff0c;这个2万依然是欠的&#xff0c;所以和8万的符号相同。</p>
