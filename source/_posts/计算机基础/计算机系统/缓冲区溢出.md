# 缓冲区溢出







----

我已经编写了一段 C 代码，并对其进行了反汇编并阅读了寄存器以了解程序在汇编中的工作方式。

```
int test(char *this){
    char sum_buf[6];
    strncpy(sum_buf,this,32);
    return 0;
}
```


我一直在检查的一段代码是测试函数。当我反汇编输出时，我的测试函数得到...

```
   0x00000000004005c0 <+12>:        mov    %fs:0x28,%rax
=> 0x00000000004005c9 <+21>:        mov    %rax,-0x8(%rbp)
... stuff ..
   0x00000000004005f0 <+60>:        xor    %fs:0x28,%rdx
   0x00000000004005f9 <+69>:        je     0x400600 <test+76>
   0x00000000004005fb <+71>:        callq  0x4004a0 <__stack_chk_fail@plt>
   0x0000000000400600 <+76>:        leaveq 
   0x0000000000400601 <+77>:        retq 
```


我想知道的是什么`mov %fs:0x28,%rax`真的在做什么？



**最佳答案**

两者`FS`和 `GS`寄存器可用作基指针地址，以便访问特殊的操作系统数据结构。所以你看到的是一个加载的值，它与 `FS` 中保存的值有一个偏移量。寄存器，而不是对 `FS` 内容的位操作登记。
具体发生了什么，是`FS:0x28`在 Linux 上存储一个特殊的哨兵堆栈保护值，并且代码正在执行堆栈保护检查。例如，如果您进一步查看代码，您将看到 `FS:0x28` 处的值。存储在堆栈中，然后调用堆栈中的内容并返回 `XOR`使用 `FS:0x28` 处的原始值执行.如果两个值相等，则说明零位已被设置，因为`XOR` 'ing 两个相同的值导致一个零值，然后我们跳转到 `test`例程，否则我们跳转到一个特殊的函数，该函数指示堆栈以某种方式损坏，并且存储在堆栈中的标记值已更改。
如果使用 GCC，[this can be disabled with](https://stackoverflow.com/a/2340455/124486) :

```
-fno-stack-protector
```
